[english] I need a function that retrieves basic hardware information like the operating system, total available memory in GB, and the actual physical CPU core count of my machine.
[english] To gather this data efficiently, I'll define a function using `platform` to determine the OS and `psutil` to query the system resources. We specifically configure the CPU check to ignore hyperthreading for an accurate physical count.
<switch:python>
[python] def local_hardware_info():
    """Basic hardware information about the local machine.

    Gives actual number of CPU's in the machine, even when hyperthreading is
    turned on. CPU count defaults to 1 when true count can't be determined.

    Returns:
        dict: The hardware information.
    """
    results = {
        'os': platform.system(),
        'memory': psutil.virtual_memory().total / (1024 ** 3),
        'cpus': psutil.cpu_count(logical=False) or 1
    }
    return results
<switch:english>
[english] The resulting dictionary provides a clean snapshot of the local machine's specs, although its functionality is entirely dependent on having both platform and psutil installed.
[english] What happens if I call this function without having imported `platform` or `psutil` at the beginning?
[english] If you haven't imported them, Python will raise a `NameError`. You must include both imports before defining the function:
<switch:python>
[python] import platform
import psutil
