[english] How can I create a decorator that transparently adds line and column metadata to AST nodes generated by my custom Python functions?
[english] This high-order function acts as a decorator factory. It takes your existing generator and returns a wrapper that intercepts the generated AST node, allowing us to attach location data before it is returned.
<switch:python>
[python] def _with_ast_loc(f):
    """Wrap a generator function in a decorator to supply line and column
    information to the returned Python AST node. Dependency nodes will not
    be hydrated, functions whose returns need dependency nodes to be
    hydrated should use `_with_ast_loc_deps` below."""

    @wraps(f)
    def with_lineno_and_col(
        ctx: GeneratorContext, node: Node, *args, **kwargs
    ) -> GeneratedPyAST:
        py_ast = f(ctx, node, *args, **kwargs)
        return _ast_with_loc(py_ast, node.env)

    return with_lineno_and_col
<switch:english>
[english] We use `node.env` within the wrapper to access the original environment context. Note that this function explicitly relies on external metadata being present in `node.env` for successful location annotation.
[english] If the input node's environment (`node.env`) doesn't contain a line and column tuple, what happens inside `_ast_with_loc`?
[english] The function will likely pass None or default values to the location fields of the AST; you must ensure that the calling context provides complete positional data.
