[english] How can I dynamically register custom functions from external plugins into my application environment?
[english] This method iterates through registered system entry points specific to 'gears:register'. It checks against an exclusion list and then executes any allowed plugin function by loading it dynamically.
<switch:python>
[python] def register_entry_points(self, exclude=()):
        """Allow Gears plugins to inject themselves to the environment. For
        example, if your plugin's package contains such ``entry_points``
        definition in ``setup.py``, ``gears_plugin.register`` function will be
        called with current environment during ``register_entry_points`` call::

            entry_points = {
                'gears': [
                    'register = gears_plugin:register',
                ],
            }

        Here is an example of such function::

            def register(environment):
                assets_dir = os.path.join(os.path.dirname(__file__), 'assets')
                assets_dir = os.path.absolute_path(assets_dir)
                environment.register(FileSystemFinder([assets_dir]))

        If you want to disable this behavior for some plugins, list their
        packages using ``exclude`` argument::

            environment.register_entry_points(exclude=['plugin'])
        """
        for entry_point in iter_entry_points('gears', 'register'):
            if entry_point.module_name not in exclude:
                register = entry_point.load()
                register(self)

[english] If a specific plugin fails during the registration process, will that failure stop all other plugins from being executed?
[english] The current implementation lacks explicit try-except blocks; therefore, an unhandled exception in one entry point's function would halt execution for subsequent ones. You might wrap the call in a try block if you want to guarantee all modules run.
