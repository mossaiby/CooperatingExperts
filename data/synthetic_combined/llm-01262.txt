[english] How can I reliably retrieve a specific column object from a set of columns, even if the input name uses different casing?
[english] To achieve case-insensitive retrieval, we first normalize the user's input by converting it to lowercase. The function then iterates through all available columns (`self.columns`) and checks each one against this normalized name.
<switch:python>
[python] def get(self, column_name):
        """
        Retrieve a column from the list with name value :code:`column_name`

        :param str column_name: The name of the column to get
        :return: :class:`~giraffez.types.Column` with the specified name, or :code:`None` if it does not exist.
        """
        column_name = column_name.lower()
        for c in self.columns:
            if c.name == column_name:
                return c
        return None
<switch:english>
[english] The function successfully returns the matching object or `None`. It's important to note that this method only performs a case-insensitive check on the name, not other potential attributes.
[english] What happens if two different columns accidentally share the exact same lowercased name?
[english] If duplicate names exist in `self.columns`, the function will return the very first column it encounters that matches the case-insensitive criteria based on iteration order.
