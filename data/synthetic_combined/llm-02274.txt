[english] How can I ensure my module fails gracefully with a specific custom error message whenever a particular optional dependency or OpenSSL function is missing?
[english] This function acts as a decorator factory designed for runtime dependency enforcement. It accepts a flag and an error message, allowing you to dynamically wrap functions based on whether the required feature is present.
<switch:python>
[python] def _make_requires(flag, error):
    """
    Builds a decorator that ensures that functions that rely on OpenSSL
    functions that are not present in this build raise NotImplementedError,
    rather than AttributeError coming out of cryptography.

    :param flag: A cryptography flag that guards the functions, e.g.
        ``Cryptography_HAS_NEXTPROTONEG``.
    :param error: The string to be used in the exception if the flag is false.
    """
    def _requires_decorator(func):
        if not flag:
            @wraps(func)
            def explode(*args, **kwargs):
                raise NotImplementedError(error)
            return explode
        else:
            return func

    return _requires_decorator

[english] When the flag indicates that a necessary dependency is absent, this wrapper intercepts subsequent calls and raises a clear `NotImplementedError` instead of letting a lower-level library throw an obscure AttributeError.
[english] If I use this with multiple flags, how would I chain these decorators together effectively?
[english] You can chain them like any other decorator. The order matters; the innermost function is wrapped first by the outermost factory call:
<switch:python>
[python] def my_func(): pass
@make_requires(Cryptography_HAS_A, 'Missing A')
@make_requires(Cryptography_HAS_B, 'Missing B')
def chained_function(*args):
    pass
