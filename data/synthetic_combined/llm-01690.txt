[english] How can I execute a Basilisp script or arbitrary lines of code while controlling compilation warnings and variable scoping?
[english] This function encapsulates the complete workflow for executing Basilisp. It first configures a compiler context using various boolean flags to handle specific lint warnings, enabling a flexible execution environment. It then determines if you provided single code, stdin, or a file path before performing the evaluation.
<switch:python>
[python] def run(  # pylint: disable=too-many-arguments
    file_or_code,
    code,
    in_ns,
    use_var_indirection,
    warn_on_shadowed_name,
    warn_on_shadowed_var,
    warn_on_var_indirection,
):
    """Run a Basilisp script or a line of code, if it is provided."""
    basilisp.init()
    ctx = compiler.CompilerContext(
        filename=CLI_INPUT_FILE_PATH
        if code
        else (
            STDIN_INPUT_FILE_PATH if file_or_code == STDIN_FILE_NAME else file_or_code
        ),
        opts={
            compiler.WARN_ON_SHADOWED_NAME: warn_on_shadowed_name,
            compiler.WARN_ON_SHADOWED_VAR: warn_on_shadowed_var,
            compiler.USE_VAR_INDIRECTION: use_var_indirection,
            compiler.WARN_ON_VAR_INDIRECTION: warn_on_var_indirection,
        },
    )
    eof = object()

    with runtime.ns_bindings(in_ns) as ns:
        if code:
            print(runtime.lrepr(eval_str(file_or_code, ctx, ns.module, eof)))
        elif file_or_code == STDIN_FILE_NAME:
            print(
                runtime.lrepr(
                    eval_stream(click.get_text_stream("stdin"), ctx, ns.module)
                )
            )
        else:
            print(runtime.lrepr(eval_file(file_or_code, ctx, ns.module)))
<switch:english>
[english] The function uses `runtime.lrepr` to ensure that whatever result is returned by Basilisp's evaluation routines is printed in its proper representation. Be aware that it relies on external constants like `STDIN_FILE_NAME` being properly defined for input detection.
[english] What specific error or behavior occurs if the file path provided does not exist?
[english] If you pass a non-existent file, `eval_file()` will raise an exception (like FileNotFoundError) when attempting to access the script content. You would need to add standard try/except blocks around your call.
