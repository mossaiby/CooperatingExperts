[english] How can I automatically find and remove very small, disconnected islands or components from my 3D mesh?
[english] This function acts as a simple wrapper to leverage existing mesh selection scripts to handle component detection. It relies on defining what 'small' means using a ratio of faces.
<switch:python>
[python] def small_parts(script, ratio=0.2, non_closed_only=False):
    """ Select & delete the small disconnected parts (components) of a mesh.

    Args:
        script: the FilterScript object or script filename to write
            the filter to.
        ratio (float): This ratio (between 0 and 1) defines the meaning of
            'small' as the threshold ratio between the number of faces of the
            largest component and the other ones. A larger value will select
            more components.
        non_closed_only (bool): Select only non-closed components.

    Layer stack:
        No impacts

    MeshLab versions:
        2016.12
        1.3.4BETA
    """
    select.small_parts(script, ratio, non_closed_only)
    selected(script)
    return None
<switch:english>
[english] The function uses the specified `ratio` against the largest component's face count to determine if smaller parts should be selected for deletion.
[english] What happens if I set the `non_closed_only` parameter first, and then specify a very low 'ratio'?
[english] If you specify `non_closed_only`, only unsealed components are considered. A low ratio like 0.01 would prioritize removing extremely tiny islands compared to larger ones.
