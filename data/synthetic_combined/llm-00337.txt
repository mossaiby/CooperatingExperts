[english] How can I dynamically create a fully qualified URI from my object attributes if one hasn't been persisted previously?
[english] This function generates a standardized identifier by first combining the collection name, item name, and tag into a basic structure. It then checks for a version attribute and conditionally appends it to complete the final URI.
<switch:python>
[python] def get_uri(self):
        '''generate a uri on the fly from database parameters if one is not
        saved with the initial model (it should be, but might not be possible)
        '''
        uri = "%s/%s:%s" %(self.collection.name, self.name, self.tag)
        if self.version not in [None,'']:
            uri = "%s@%s" %(uri, self.version)
        return uri
<switch:english>
[english] The method relies on standard string formatting (`%s`) to construct the URI from available attributes. However, users should be aware that this implementation does not handle complex URL encoding for special characters in names or tags.
[english] If the `self.tag` contains reserved URI characters like slashes or question marks, will those characters break the generated URI?
[english] Yes, because it uses simple string concatenation without specialized encoding (like `urllib.parse.quote`), any reserve characters could misinterpret the resource location.
