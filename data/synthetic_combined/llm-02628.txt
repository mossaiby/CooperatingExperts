[english] How can I gracefully cancel a `task_id` using the scheduler and handle cases where the task might not exist?
[english] To safely remove a task, we attempt to use the internal scheduler's cancellation method inside a try block. We wrap this logic with exception handling so that if the specific task ID isn't found, we log the event instead of crashing.

[python] def remove_task(self, task_id):
        """Remove and cancel a task.

        :param task_id: id of the task to be removed
        """
        try:
            self._scheduler.cancel_task(task_id)
        except NotFoundError as e:
            logger.info("Cannot cancel %s task because it does not exist.",
                        task_id)
            return False
        else:
            return True
