[english] How can I generate the correct API endpoint URL for retrieving stock price data when I have both a ticker symbol and the required update frequency?
[english] This function acts as an internal utility to route requests correctly. It first ensures the input frequency is acceptable, then determines whether the request should use the Tiingo API for End-of-Day (EOD) reports or the IEX API for intraday usage.
<switch:python>
[python] def _get_url(self, ticker, frequency):
        """
        Return url based on frequency.  Daily, weekly, or yearly use Tiingo
        EOD api; anything less than daily uses the iex intraday api.
        :param ticker (string): ticker to be embedded in the url
        :param frequency (string): valid frequency per Tiingo api
        :return (string): url
        """
        if self._invalid_frequency(frequency):
            etext = ("Error: {} is an invalid frequency.  Check Tiingo API documentation "
                     "for valid EOD or intraday frequency format.")
            raise InvalidFrequencyError(etext.format(frequency))
        else:
            if self._is_eod_frequency(frequency):
                return "tiingo/daily/{}/prices".format(ticker)
            else:
                return "iex/{}/prices".format(ticker)

[english] Note that the URL construction relies entirely on two helper methods (`self._invalid_frequency` and `self._is_eod_frequency`) to accurately categorize the input frequency before proceeding.
