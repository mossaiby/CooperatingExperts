[english] How can I implement a function that reliably fetches textures by name and category while efficiently managing them using a cache?
[english] This method prioritizes speed by first checking the `categoriesTexCache`. If the target category is missing entirely, it attempts to retrieve it before looking for the specific texture within that category.
<switch:python>
[python] def getTex(self,name,category):
        """
        Gets the texture associated with the given name and category.
        
        ``category`` must have been created using :py:meth:`addCategory()` before.
        
        If it was loaded previously, a cached version will be returned.
        If it was not loaded, it will be loaded and inserted into the cache.
        
        See :py:meth:`loadTex()` for more information.
        """
        if category not in self.categoriesTexCache:
            return self.getMissingTex(category)
        if name not in self.categoriesTexCache[category]:
            self.loadTex(name,category)
        return self.categoriesTexCache[category][name]
<switch:english>
[english] The logic ensures that textures are loaded into the cache (`self.categoriesTexCache`) automatically if they are accessed for the first time.
[english] What happens if I call `getTex` with a category name that was never added via `addCategory()`?
[english] If the category is truly unsupported, it will be passed to `self.getMissingTex(category)`. This function handles the case where the structure it relies on isn't initialized.
