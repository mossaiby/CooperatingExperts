[english] How can I reliably fetch a configuration option from a section while ensuring a default value is set if it doesn't already exist?
[english] This method employs a try/except block to handle missing settings gracefully. It first attempts to retrieve the stored value, and only falls back to setting the provided value if the retrieval fails.
<switch:python>
[python] def get_or_set_default(self, section, option, value):
        """
        Base method to fetch values and to set defaults in case they
        don't exist.
        """
        try:
            ret = self.get(section, option)
        except MissingSetting:
            self.set(section, option, value)
            ret = value

        return ret
<switch:english>
[english] This design pattern guarantees that the requested key will always have a return value while preventing unnecessary overwrites of existing settings.
[english] What happens if I want to ensure an existing numerical setting is cast into integer format, even if it was stored as a string?
[english] The function itself doesn't handle type casting. You would need to add logic after retrieval (or catching the default) to explicitly convert `ret = int(value)` before returning.
