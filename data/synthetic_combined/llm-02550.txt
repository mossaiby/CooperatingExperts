[english] How can I create a helper to fetch configuration values from an active connection's settings first, and then default to a global set of application parameters?
[english] This approach uses conditional logic to check for the key in a local dictionary before falling back. If the key isn't present there, it defaults to retrieving the value directly from the global `settings` object.

[python] def get_setting(connection, key):
    """Get key from connection or default to settings."""
    if key in connection.settings_dict:
        return connection.settings_dict[key]
    else:
        return getattr(settings, key)

[english] The code prioritizes connection-specific settings while maintaining a centralized default lookup mechanism via `getattr`. Note that `getattr` will raise an error if the key is unavailable everywhere else.
[english] What happens exactly if the requested 'key' does not exist in either the connection or the global settings?
