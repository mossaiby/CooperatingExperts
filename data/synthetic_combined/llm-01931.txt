[english] How can I adjust a signal's bit width—by slicing or arithmetic extension—to precisely match the required size of a target data type?
[english] This function calculates the difference between the input signal's width and the target type's width. It then either slices the signal if it is too large or extends it using proper sign-extension logic.
<switch:python>
[python] def fitTo_t(what: Union[RtlSignal, Value], where_t: HdlType,
            extend: bool=True, shrink: bool=True):
    """
    Slice signal "what" to fit in "where"
    or
    arithmetically (for signed by MSB / unsigned, vector with 0) extend
    "what" to same width as "where"

    little-endian impl.
    """

    whatWidth = what._dtype.bit_length()
    toWidth = where_t.bit_length()
    if toWidth == whatWidth:
        return what
    elif toWidth < whatWidth:
        # slice
        if not shrink:
            raise BitWidthErr()

        return what[toWidth:]
    else:
        if not extend:
            raise BitWidthErr()

        w = toWidth - whatWidth

        if what._dtype.signed:
            # signed extension
            msb = what[whatWidth - 1]
            ext = reduce(lambda a, b: a._concat(b), [msb for _ in range(w)])
        else:
            # 0 extend
            ext = vec(0, w)

        return ext._concat(what)

[english] The implementation handles two main cases: slicing when shrinking, or arithmetically extending based on whether the original data was signed or unsigned.
[english] If the input signal width is smaller than `where_t` but we set `extend=False`, does this function still perform an extension?
[english] No, if `extend=False` and a wider target type is requested, the function will raise a BitWidthErr() to prevent unintended data loss.
