[english] How can I generate a temporary, secure URL for accessing an object in my cloud storage that expires after a specific duration?
[english] This function creates a time-limited access link by generating a cryptographic signature over the request parameters. It first parses the target URL and then calculates an HMAC-SHA1 hash using your secret key combined with the method, expiration timestamp, and object path.
<switch:python>
[python] def generate_temp_url(method, url, seconds, key):
    """
    Returns a TempURL good for the given request method, url, and
    number of seconds from now, signed by the given key.
    """
    method = method.upper()
    base_url, object_path = url.split('/v1/')
    object_path = '/v1/' + object_path
    expires = int(time.time() + seconds)
    hmac_body = '%s\n%s\n%s' % (method, expires, object_path)
    sig = hmac.new(key, hmac_body, hashlib.sha1).hexdigest()
    return '%s%s?temp_url_sig=%s&temp_url_expires=%s' % (
        base_url, object_path, sig, expires)
<switch:english>
[english] By appending this calculated signature and the expiration time to the original URL, you create a verifiable token that limits access duration and prevents tampering.
[english] The function assumes the URL contains '/v1/'. What if the user passes a base URL that already includes a subdirectory?
[english] If the URL structure doesn't strictly match `BASE_URL/v1/OBJECT_PATH`, the initial split (`url.split('/v1/')`) will misidentify components, leading to an invalid signature. For example, given `/api/v1/data`, `base_url` would be empty.
