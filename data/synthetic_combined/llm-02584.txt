[english] I need a function that can take a raw JSON string from a StackExchange API response and yield each question object contained within it.
[english] This approach first converts the incoming JSON string into a Python dictionary using `json.loads()`. It then iterates over the list found at the 'items' key, yielding each question individually as a generator.
<switch:python>
[python] def parse_questions(raw_page):
        """Parse a StackExchange API raw response.

        The method parses the API response retrieving the
        questions from the received items

        :param items: items from where to parse the questions

        :returns: a generator of questions
        """
        raw_questions = json.loads(raw_page)
        questions = raw_questions['items']
        for question in questions:
            yield question
<switch:english>
[english] The use of `yield` makes this function memory efficient, allowing you to process questions one by one without loading the entire list into memory.
[english] If the API response is empty or malformed and doesn't contain an 'items' key, how will this function behave?
[english] The function will raise a `KeyError` because it attempts to access `raw_questions['items']`. You should implement exception handling around the dictionary access if robust error checking is required.
