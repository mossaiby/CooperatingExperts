[english] How can I programmatically find the resulting broadcast shape for a list of input tensors that might have unknown dimensions during graph construction?
[english] This function tries to be efficient by calculating the shapes using static methods first rather than sacrificing performance at runtime. If static checks fail because some dimensions are undefined, it falls back to a dynamic approach.

[python] def get_broadcast_shape(*tensors):
  """Get broadcast shape as a Python list of integers (preferred) or `Tensor`.

  Args:
    *tensors:  One or more `Tensor` objects (already converted!).

  Returns:
    broadcast shape:  Python list (if shapes determined statically), otherwise
      an `int32` `Tensor`.
  """
  # Try static.
  s_shape = tensors[0].shape
  for t in tensors[1:]:
    s_shape = tf.broadcast_static_shape(s_shape, t.shape)
  if tensorshape_util.is_fully_defined(s_shape):
    return tensorshape_util.as_list(s_shape)

  # Fallback on dynamic.
  d_shape = tf.shape(input=tensors[0])
  for t in tensors[1:]:
    d_shape = tf.broadcast_dynamic_shape(d_shape, tf.shape(input=t))
  return d_shape

[english] The first path returns Python list of integers when possible, but if any shape is not fully defined, it will return a symbolic `Tensor`. This allows the function to handle both graph-mode and eager execution efficiently.
[english] What happens if one of the input tensors has an unknown rank (e.g., unknown number of dimensions)?
[english] Static shape calculations require known ranks. If the rank is not fully defined, the dynamic fallback uses `tf.shape(input=t)` on each tensor to determine its runtime dimensionality.
