[english] How can I securely validate an authorization header token sent by the Bot Framework and extract its identity claims?
[english] This function performs asynchronous token authentication to verify that an incoming request truly originated from the Bot Framework service. It validates the token using provided credentials and then strictly checks a required internal claim against a trusted application property.
<switch:python>
[python] async def authenticate_token_service_url(auth_header: str, credentials: CredentialProvider, service_url: str, channel_id: str) -> ClaimsIdentity:
        """ Validate the incoming Auth Header

        Validate the incoming Auth Header as a token sent from the Bot Framework Service.
        A token issued by the Bot Framework emulator will FAIL this check.

        :param auth_header: The raw HTTP header in the format: 'Bearer [longString]'
        :type auth_header: str
        :param credentials: The user defined set of valid credentials, such as the AppId.
        :type credentials: CredentialProvider
        :param service_url: Claim value that must match in the identity.
        :type service_url: str

        :return: A valid ClaimsIdentity.
        :raises Exception:
        """
        identity = await asyncio.ensure_future(
            ChannelValidation.authenticate_token(auth_header, credentials, channel_id))

        service_url_claim = identity.get_claim_value(ChannelValidation.SERVICE_URL_CLAIM)
        if service_url_claim != service_url:
            # Claim must match. Not Authorized.
            raise Exception('Unauthorized. service_url claim do not match.')

        return identity

[english] The primary takeaway is that successful validation guarantees both token authenticity and that the service URL within the identity matches your configuration exactly before returning it.
[english] If there is no claim value found for `ChannelValidation.SERVICE_URL_CLAIM`, will this function raise an error before the comparison check?
[english] The internal token authentication logic usually ensures necessary claims exist. If a claim is missing, attempting to access or compare against `None` will cause the comparison step to fail, raising the 'Unauthorized' exception.
