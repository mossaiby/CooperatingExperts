[english] How do I create an endpoint decorator that registers a view function into a state without automatically prefixing it with the current blueprint name?
[english] This method acts as a factory, returning a decorator that handles the registration of your viewed function using `self.record_once`. Inside this pattern, we define how and when the endpoint is actually mapped to the global application view functions.
<switch:python>
[python] def endpoint(self, endpoint):
        """Like :meth:`Flask.endpoint` but for a blueprint.  This does not
        prefix the endpoint with the blueprint name, this has to be done
        explicitly by the user of this method.  If the endpoint is prefixed
        with a `.` it will be registered to the current blueprint, otherwise
        it's an application independent endpoint.
        """
        def decorator(f):
            def register_endpoint(state):
                state.app.view_functions[endpoint] = f
            self.record_once(register_endpoint)
            return f
        return decorator

[english] The implementation uses a nested registration function that is executed by `self.record_once`, ensuring the view function `f` is mapped to the specific `endpoint` key during initialization.
