[english] How can I ensure that any datetime retrieved from our database is consistently converted and represented in Coordinated Universal Time (UTC)?
[english] This function standardizes datetime objects by ensuring they are wrapped or converted into the UTC timezone, which helps create consistent data representation.
<switch:python>
[python] def process_result_value(self, value, dialect):
        """
        Processes DateTimes from the DB making sure it is always
        returning UTC. Not using timezone.convert_to_utc as that
        converts to configured TIMEZONE while the DB might be
        running with some other setting. We assume UTC datetimes
        in the database.
        """
        if value is not None:
            if value.tzinfo is None:
                value = value.replace(tzinfo=utc)
            else:
                value = value.astimezone(utc)

        return value

[english] A key consideration is that if the input `value` is naive (lacks timezone info), the function assumes it was recorded in UTC rather than attempting to guess its original source timezone.
[english] What happens if the datetime object already contains a non-UTC timezone that we want to preserve without conversion?
[english] The current implementation will always convert any existing aware datetime objects (those with `tzinfo`) into UTC using `.astimezone(utc)`. If you needed to preserve other timezones, the logic would require modification.
