[english] How can I implement a logic that resolves whether a current execution stack frame matches a breakpoint, which might be set by a specific line number or a function's name?
[english] This helper function determines if a debugger should pause execution, comparing the provided breakpoint criteria (b) against the active call context (frame). It first checks if the breakpoint was configured using a specific line number.
<switch:python>
[python] def checkfuncname(b, frame):
    """Check whether we should break here because of `b.funcname`."""
    if not b.funcname:
        # Breakpoint was set via line number.
        if b.line != frame.f_lineno:
            # Breakpoint was set at a line with a def statement and the function
            # defined is called: don't break.
            return False
        return True

    # Breakpoint set via function name.

    if frame.f_code.co_name != b.funcname:
        # It's not a function call, but rather execution of def statement.
        return False

    # We are in the right frame.
    if not b.func_first_executable_line:
        # The function is entered for the 1st time.
        b.func_first_executable_line = frame.f_lineno

    if b.func_first_executable_line != frame.f_lineno:
        # But we are not at the first line number: don't break.
        return False
    return True
<switch:english>
[english] For breakpoints set by function name, the code ensures it only triggers once upon entry by checking `b.func_first_executable_line`. This prevents redundant breaks during subsequent executions of the same function.
[english] If a function is called multiple times later in its lifecycle, will this breakpoint continue to fire?
[english] No, because if `b.func_first_executable_line` has been set (meaning the function was entered before), and it does not match the current line number (`frame.f_lineno`), the function returns False.
