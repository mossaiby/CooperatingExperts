[english] How can I register an error handler specifically to catch certain HTTP status codes or exceptions generated by views inside a Flask Blueprint?
[english] This utility is implemented as a decorator factory which allows you to define custom logic for errors within the current blueprint's scope. It uses internal methods on the `self` object to ensure that handlers are registered only once with the parent application if they match specific codes or exceptions.
<switch:python>
[python] def errorhandler(self, code_or_exception):
        """Registers an error handler that becomes active for this blueprint
        only.  Please be aware that routing does not happen local to a
        blueprint so an error handler for 404 usually is not handled by
        a blueprint unless it is caused inside a view function.  Another
        special case is the 500 internal server error which is always looked
        up from the application.

        Otherwise works as the :meth:`~flask.Flask.errorhandler` decorator
        of the :class:`~flask.Flask` object.
        """
        def decorator(f):
            self.record_once(lambda s: s.app._register_error_handler(
                self.name, code_or_exception, f))
            return f
        return decorator
<switch:english>
[english] While this method localizes error handler definition, be aware that routing is global; this handler primarily intercepts errors raised by views rather than non-routed requests like a 404. This code ensures the registration process for your custom error handlers runs cleanly.
[english] Since Blueprint routing is not entirely isolated, can I expect an error handler defined here to globally override or participate in catch blocks set up higher in my application structure?
[english] Because of how it calls `app._register_error_handler`, the blueprint handler gets registered with the main application instance internally. This allows it to participate in a chain, but often requires careful ordering if you intend for this Blueprint's logic to take precedence over global handlers.
