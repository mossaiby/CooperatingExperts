[english] How can I retrieve a list of all existing user-defined tables within my SQLite database?
[english] To find table names, we query the system catalog (`sqlite_master`) to select entries where the type is 'table'. We then collect these into a list, optionally filtering out internal system tables.
<switch:python>
[python] def fetch_table_names(self, include_system_table=False):
        """
        :return: List of table names in the database.
        :rtype: list
        """

        result = self.__cur.execute("SELECT name FROM sqlite_master WHERE TYPE='table'")
        if result is None:
            return []

        table_names = [record[0] for record in result.fetchall()]

        if include_system_table:
            return table_names

        return [table for table in table_names if table not in SQLITE_SYSTEM_TABLES]

[english] The function handles potential missing results by checking for `None` after execution and gracefully returns an empty list if no data is found. Note that external dependencies on global variables like `SQLITE_SYSTEM_TABLES` must be carefully managed.
[english] What happens to the table names returned if the function encounters multiple types of records besides 'table' in the system catalog?
[english] The current query explicitly filters using `WHERE TYPE='table'`, so it only returns database objects designated as tables. If other types were present, they would be ignored by the SQL statement.
