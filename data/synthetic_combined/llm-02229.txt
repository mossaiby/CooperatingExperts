[english] Currently, the function assumes success and blindly calls `r.text`, which might raise an exception if the request failed dramatically. To handle this, you should check the response status code:
<switch:python>
[python] def __send_request(self, url, params=None):
    r = self.fetch(url, payload=params)
    if r.status_code != 200:
        raise ConnectionError(f"Request failed with status {r.status_code}")
    return r.text
