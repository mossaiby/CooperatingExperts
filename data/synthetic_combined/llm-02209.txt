[english] This function determines if a provided `path` matches a defined URL configuration. It first resolves the path using `urlresolvers.resolve`, then verifies both the unique internal URL name and validates every supplied argument against what the resolver expects.
<switch:python>
[python] def pathMatches(path, urlName, **kwargs):
	'''
	:param path: str
	:param urlName: str
	:returns: bool.
	'''
	resolved = urlresolvers.resolve(path)

	# Different URL name => the current URL cannot match.
	resolvedName = '{r.namespace}:{r.url_name}'.format(r = resolved) if resolved.namespace else resolved.url_name
	if urlName != resolvedName:
		return False

	# If any of the current keyword args is missing or different, the URL cannot match.
	for key, value in kwargs.items():
		currentArg = resolved.kwargs.get(key)
		if currentArg is None or str(value) != str(currentArg):
			return False

	return True

[english] The final check ensures consistency by comparing all keyword arguments after converting them to strings, assuming that parameter values should match exactly regardless of original type.
[english] How does this function handle cases where a required parameter defined in the path mapping is completely missing from the `**kwargs` input?
[english] If an expected argument is missing from the input `kwargs`, `resolved.kwargs.get(key)` will return None, and the function correctly fails by triggering the check `if currentArg is None`. This guarantees that all required parameters must be explicitly provided.
