[english] How can I create an exact copy of an existing ParseNode but change its node type?
[english] Since all nodes share properties like consumed and position, this function builds a new instance using your current data but maps in the desired new type.
<switch:python>
[python] def retyped(self, new_type):
    """Returns a new node with the same contents as self, but with a new node_type."""
    return ParseNode(new_type,
                      children=list(self.children),
                      consumed=self.consumed,
                      position=self.position,
                      ignored=self.ignored)
<switch:english>
[english] Note that by creating and returning a new instance instead of modifying the original object in place, you ensure immutability for this specific change.
[english] If 'children' is mutable, should I deep copy it to prevent side effects?
[english] Because children are listed using `list(self.children)`, only a shallow copy is made. If your node contains nested complex structures (like another ParseNode) that you want completely independent of the original, you would need to recursive dypcopy them.
