[english] How do I format a custom metric into a standardized string compatible with StatsD ingestion tools?
[english] This function standardizes metrics for compatibility by building a template and then conditionally prepending any defined service prefix to the metric name.
<switch:python>
[python] def format_metric_string(self, name, value, m_type):
        """Compose a statsd compatible string for a metric's measurement."""

        # NOTE(romcheg): This serialized metric template is based on
        #                statsd's documentation.
        template = '{name}:{value}|{m_type}\n'

        if self.prefix:
            name = "{prefix}.{m_name}".format(prefix=self.prefix, m_name=name)

        return template.format(name=name, value=value, m_type=m_type)
