[english] How can I correctly convert constant Lisp forms into generated Python AST nodes?
[english] This function acts as a dispatcher, looking up a specific handler based on the type of the input form to perform the necessary transformation.
<switch:python>
[python] def _const_val_to_py_ast(ctx: GeneratorContext, form: LispForm) -> GeneratedPyAST:
    """Generate Python AST nodes for constant Lisp forms.

    Nested values in collections for :const nodes are not parsed, so recursive
    structures need to call into this function to generate Python AST nodes for
    nested elements. For top-level :const Lisp AST nodes, see
    `_const_node_to_py_ast`."""
    handle_value = _CONST_VALUE_HANDLERS.get(type(form))
    if handle_value is None and isinstance(form, ISeq):
        handle_value = _const_seq_to_py_ast  # type: ignore
    assert handle_value is not None, "A type handler must be defined for constants"
    return handle_value(ctx, form)

[english] It relies entirely on the `_CONST_VALUE_HANDLERS` registry; if a required converter is missing, an assertion error will be raised.
[english] What happens if a custom Lisp object isn't explicitly registered in `_CONST_VALUE_HANDLERS`?
[english] If the form type does not have a corresponding handler and is not a sequence (`ISeq`), the function asserts that a handler must exist, causing it to fail.
