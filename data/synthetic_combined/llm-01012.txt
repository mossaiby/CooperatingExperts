[english] How can I configure an object to read data from a specific file path or an existing stream?
[english] To make this function, we check the type of provided input first. If it's already a text-based IO stream, we reuse it; otherwise, if it's a string representing a path, we open and wrap that file.

[english] This method provides flexibility by accepting either an existing reader or a path string. However, the check for strings relies on inspecting the internal classes of 'string' module objects, which can sometimes be brittle.
[english] What happens if I try to open a directory path instead of a file?
[english] If you pass a directory, Python will raise an `OSError` (like `FileNotFoundError`) during the execution of the standard `open(inp, 'r')` call.
