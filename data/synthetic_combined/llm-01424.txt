[english] How can I retrieve a list of all users belonging to this Slack team efficiently?
[english] This function uses lazy loading and caching to ensure that the potentially slow API call is only made once, even if the method is called multiple times. It first checks an internal attribute to see if the required user data is already available.
<switch:python>
[python] def users(self):
        """
        List of users of this slack team
        """
        if not self._users:
            self._users = self._call_api('users.list')['members']
        return self._users
<switch:english>
[english] This pattern makes subsequent calls extremely fast as it relies on the stored value, avoiding redundant network requests. However, be aware that this cached list will not automatically update if a member joins or leaves the team externally.
[english] What happens if the external API call fails or returns data in an unexpected format?
[english] If `self._call_api` raises an exception during execution, the error will propagate up. If it successfully calls the API but finds no members, `self._users` is set to that empty list and returned immediately.
