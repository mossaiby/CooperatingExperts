[english] I need a helper method that calculates the precise string of spaces and tabs required to align a specific token correctly on its line.
[english] This approach first retrieves the base indentation for the current line using `self.line_indent`. Then, it calculates the necessary padding by subtracting the existing indent length from the token's starting column.
<switch:python>
[python] def token_indent(self, idx):
        """Get an indentation string for hanging indentation, consisting of the line-indent plus
        a number of spaces to fill up to the column of this token.

        e.g. the token indent for foo
        in "<TAB><TAB>print(foo)"
        is "<TAB><TAB>      "
        """
        line_indent = self.line_indent(idx)
        return line_indent + " " * (self.start_col(idx) - len(line_indent))
<switch:english>
[english] The function guarantees that the returned string, when prepended to the token, places it exactly at its calculated starting column.
[english] What happens if the token's start column is less than or equal to the length of the line indent?
[english] If `self.start_col(idx)` is too small, the subtraction results in zero or a negative number. Since string multiplication does not allow negative counts, Python will effectively return an empty space string rather than crashing.
