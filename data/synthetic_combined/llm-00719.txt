[english] I need a function that accepts a local zip file path and uploads it as source code to Google Cloud Functions.
[english] This method first queries the GCP API to get an upload URL for the function deployment, and then uses the `requests` library to stream the content of the specified local zip file directly to that URL.
<switch:python>
[python] def upload_function_zip(self, location, zip_path, project_id=None):
        """
        Uploads zip file with sources.

        :param location: The location where the function is created.
        :type location: str
        :param zip_path: The path of the valid .zip file to upload.
        :type zip_path: str
        :param project_id: Optional, Google Cloud Project project_id where the function belongs.
            If set to None or missing, the default project_id from the GCP connection is used.
        :type project_id: str
        :return: The upload URL that was returned by generateUploadUrl method.
        """
        response = self.get_conn().projects().locations().functions().generateUploadUrl(
            parent=self._full_location(project_id, location)
        ).execute(num_retries=self.num_retries)
        upload_url = response.get('uploadUrl')
        with open(zip_path, 'rb') as fp:
            requests.put(
                url=upload_url,
                data=fp,
                # Those two headers needs to be specified according to:
                # https://cloud.google.com/functions/docs/reference/rest/v1/projects.locations.functions/generateUploadUrl
                # nopep8
                headers={
                    'Content-type': 'application/zip',
                    'x-goog-content-length-range': '0,104857600',
                }
            )
        return upload_url
<switch:english>
[english] The use of specific headers like 'x-goog-content-length-range' ensures compatibility with the Google Cloud storage service during the data transfer. This implementation returns the initial upload URL rather than the final function resource name.
[english] If the content upload fails midway through a large zip file, does this code handle retrying the entire operation?
[english] The method uses `num_retries` only for fetching the initial upload URL via `.execute()`. Interruptions during the actual data transfer (the `requests.put`) would require an external retry mechanism.
