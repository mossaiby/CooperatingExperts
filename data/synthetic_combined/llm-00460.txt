[english] I need to create a dictionary of context data for sending a password reset email to an authenticated user.
[english] This function is designed to package all necessary information required by your application's email service, ensuring it knows which user and site are involved in the request. It utilizes methods directly on the `notification` object to securely generate identifiers and tokens.
<switch:python>
[python] def password_reset_email_context(notification):
    """Email context to reset a user password."""
    return {
        'protocol': 'https',
        'uid': notification.user.generate_uid(),
        'token': notification.user.generate_token(),
        'site': notification.site,
    }
<switch:english>
[english] The use of `notification.user.generate_uid()` ensures that you get a consistent unique identifier while separating authentication concerns from the data packaging logic.
[english] If I want to manually specify a 'type' like 'password_reset', where would I add that field?
