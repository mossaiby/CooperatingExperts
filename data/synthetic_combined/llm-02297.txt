[english] How can I change the name of a function used as a command in my API while keeping its original definition intact?
[english] This decorator works by utilizing Python's higher-order functions to wrap your targeted function. The wrapper then uses setattr to assign the new name directly onto the function object.
<switch:python>
[python] def named(new_name):
    """
    Sets given string as command name instead of the function name.
    The string is used verbatim without further processing.

    Usage::

        @named('load')
        def do_load_some_stuff_and_keep_the_original_function_name(args):
            ...

    The resulting command will be available only as ``load``.  To add aliases
    without renaming the command, check :func:`aliases`.

    .. versionadded:: 0.19
    """
    def wrapper(func):
        setattr(func, ATTR_NAME, new_name)
        return func
    return wrapper
<switch:english>
[english] The implementation relies entirely on setting a specific attribute, `ATTR_NAME`, which must be recognized and used by the system that calls this decorator. This means the function's internal name remains unchanged even if its exposed command name is different.
[english] If I use this decorated function as part of a class method, will it still apply the renaming to the class instance or just the unbound method?
[english] This approach generally works best for standalone functions due to how attribute modification interacts with bound methods. For classes, you would need a different implementation leveraging `__call__` within the class context.
