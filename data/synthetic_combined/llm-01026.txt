[english] How do I ensure that when a child widget is removed from its parent container, it is also properly cleaned up and deleted from the internal list of all children?
[english] This method handles cleanup after a child event fires. It first calls the parent implementation via `super()`, then checks if the `child` has an associated active widget that needs removing.
<switch:python>
[python] def child_removed(self, child):
        """ Handle the child removed event from the declaration.

        This handler will unparent the child toolkit widget. Subclasses
        which need more control should reimplement this method.

        """
        super(WebComponent, self).child_removed(child)
        if child.widget is not None:
            for i, c in enumerate(self.children()):
                if c == child:
                    del self.widget[i]
                    break

[english] The function iterates through `self.children()` and uses the index to delete the item from `self.widget`, ensuring that only children with active widgets are targeted for internal removal.
